# Building This Website

**Date:** Summer 2025 - Present
**Tools:** Jekyll, GitHub Pages, HTML/CSS, Markdown

A personal portfolio site built with Jekyll and GitHub Pages to document and share my work with others.

---

## Problem and Purpose

I've created this website to document my projects and share the work I do in an intentional way. Platforms like LinkedIn prioritize polished results and performative success, which overlooks the value of ongoing work, learning, and iteration. I wanted a space where I could highlight both the process and outcomes of my projects, without feeling the pressure to create a curated feed for a social media platform.

This site allows me to showcase what I'm working on to recruiters, collaborators, and anyone interested in my technical or creative journey. My goal is to offer a transparent, professional portfolio that reflects what I've built and how I think.

---
## Milestones 

The website you're viewing is the direct result of this project. I've designed this website through the use of a template to serve as a flexible, minimal, and easy-to-maintain portfolio

**Key Contributions Include:**
- Set up a GitHub Pages repository with the [Just the Docs](https://just-the-docs.github.io/just-the-docs/) Jekyll theme
- Customized `_config.yml` for navigation and styling
- Wrote Markdown files for project documentation
- Installed and configured Ruby, Jekyll, and Git to enable local development

---

## Development Process

**Initial Set-up**: Installed Ruby, Jekyll, and Git to create a local development environment, allowing for efficient testing and content updates without relying on live deployment.

**Layout and Structure**: Download [Just the Docs](https://just-the-docs.github.io/just-the-docs/) Jekyll theme and use navigation system to organize projects into academic, personal engineering, and creative categories.

**Content Strategy**: Focused on building reusable page templates that highlight both project outcomes and development processes, while maintaining a professional tone distinct from social media platforms.

**Version Control and Deployment:** Managed site throguh Git and Github Pages, using local repositories to state and review changed before pushing them live.

**Ongoing Iteration:** Continuously refining page layouts, content organization, and page content as projects change.
