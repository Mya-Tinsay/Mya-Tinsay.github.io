# Building This Website

**Date:** Summer 2025  
**Tools:** Jekyll, GitHub Pages, HTML/CSS, Markdown

---

## 🔍 Overview

I created this personal website to showcase my engineering work, document my projects, and reflect on my learning journey. Rather than just listing finished products, I wanted to show the process behind the work I do.

---

## 🛠 Motivation

I wanted a simple, clean platform where I could present projects in a way that felt authentic—without relying on overly polished LinkedIn updates. Building the site from scratch allowed me to customize both its content and structure while learning how static site generators work.

---

## 🚧 Development Process

- Set up a GitHub Pages repository with the [Just the Docs](https://just-the-docs.github.io/just-the-docs/) Jekyll theme
- Customized `_config.yml` for navigation and styling
- Wrote Markdown files for project documentation and blog-style updates
- Used permalinks to structure URLs clearly and consistently
- Iteratively tested the site using `jekyll serve` locally before pushing updates

---

## 📈 Takeaways

- Gained a deeper understanding of static site generation and templating systems
- Improved my GitHub workflow through versioned content updates
- Learned how to organize project documentation for clarity and usability
- Built a long-term platform I can grow as I take on more projects
